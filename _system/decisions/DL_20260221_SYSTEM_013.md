# DECISION LOG ENTRY: DL_20260221_SYSTEM_013

## DECISION
Las DL entries usan el formato de nombre DL_YYYYMMDD_[SUBSYSTEM]_[NNN].md con numeración global secuencial de 3 dígitos continua entre todos los subsistemas y fechas.

## RATIONALE
La numeración global garantiza identificadores únicos sin ambigüedad. El formato con subsistema en el nombre permite saber el origen sin abrir el archivo. La secuencia global (no por subsistema) evita colisiones como RESEARCH_001 y KB_001 coexistiendo.

## AFFECTED SUBSYSTEMS
- Todos: todos los chats deben usar este formato al crear DL entries
