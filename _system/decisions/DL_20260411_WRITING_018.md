# DECISION LOG ENTRY: DL_20260411_WRITING_018

## DECISION

EDITOR_PROFILE y PUBLICATION_PROFILE son entidades independientes con ciclo de vida propio. Un editor puede escribir en varias publicaciones; una publicación puede tener varios editores. El PUBLICATION_PROFILE no vive dentro del EDITOR_PROFILE — es un documento independiente con su propio identificador único. La relación entre ambos se establece en el WRITING_CONTEXT, no por herencia estructural.

## RATIONALE

La relación entre editor y publicación es muchos a muchos: un editor publica en múltiples medios con criterios distintos; un medio puede tener múltiples editores colaboradores. Modelar el PUBLICATION_PROFILE como lista dentro del EDITOR_PROFILE rompería esa flexibilidad y obligaría a duplicar información cuando varios editores comparten una publicación.

## AFFECTED SUBSYSTEMS

- WRITING: RESOURCE_PUBLICATION_PROFILE como entidad independiente con identificador único
- EDITORIAL: TEMPLATE_EDITOR_PROFILE no incluye lista de publicaciones — la relación vive en WRITING_CONTEXT. Pendiente actualización en Sprint 4.
- EVALUATION: PROMPT_EVALUATE_POST recibe PUBLICATION_PROFILE como input independiente vía WRITING_CONTEXT
