# DECISION LOG ENTRY: DL_20260222_KB_003

## DECISION

Sección `CANONICAL UPDATE SCHEMA` añadida a `RESOURCE_SOURCE_AUTHORITY` (v2.0 → v2.1) y `RESOURCE_CLAIM_VALIDATION` (v1.0 → v1.1), estableciendo el contrato de actualización ejecutable de ambos recursos.

## RATIONALE

Ambos documentos describían su contenido pero no especificaban mecánicamente cómo actualizarlos. Sin este contrato, `PROMPT_UPDATE_VALIDATION_CHECKLIST` no puede ejecutar actualizaciones de forma autónoma y consistente.

## AFFECTED SUBSYSTEMS

- RESEARCH: `PROMPT_UPDATE_VALIDATION_CHECKLIST` debe actualizarse para leer y aplicar los procedimientos definidos en `CANONICAL UPDATE SCHEMA` de ambos recursos antes de ejecutar cualquier actualización.

## ARTIFACTS AFFECTED

| Artifact | Action | Notes |
|----------|--------|-------|
| RESOURCE_SOURCE_AUTHORITY | MODIFY v2.0 → v2.1 | Añadida sección CANONICAL UPDATE SCHEMA: field spec (campos obligatorios/opcionales, vocabulario controlado Type con 16 valores), procedimientos A–D (añadir fuente, añadir topic, reclasificar, deprecar), tabla de triggers de notificación a KB-dev. |
| RESOURCE_CLAIM_VALIDATION | MODIFY v1.0 → v1.1 | Añadida sección CANONICAL UPDATE SCHEMA: arquitectura de dos capas (Universal Framework = solo KB-dev / Topic-Specific = Research autónomo), field spec para secciones de topic, procedimientos A–D, tabla de triggers, tabla de alineación SAH↔CVC. |
| PROMPT_UPDATE_VALIDATION_CHECKLIST | MODIFY vX.Y → vX.Y+1 | Referenciar CANONICAL UPDATE SCHEMA de SAH y CVC. Aplicar procedimientos A–D según tipo de cambio. Respetar distinción de capas del CVC. Emitir DL entries en los casos indicados en las tablas de triggers. |
