# DECISION LOG ENTRY: DL_20260411_SYSTEM_021

## DECISION

El EDITOR_PROFILE se enriquece progresivamente mediante aprendizaje del sistema, no por aprobación manual del editor. Las tres fuentes de señal son: (1) output del Q&A con material marcado como señal de aprendizaje, (2) correcciones manuales del editor sobre el borrador registradas en POST_FINAL_EDITED, (3) delta entre borrador del sistema y texto publicado cuando el editor lo proporciona. Lo que actualiza el perfil son patrones estructurales estables — no temas, anécdotas ni construcciones literales concretas de un post específico. El perfil debe ser estable y de evolución lenta.

En Sprint 3 se especifican y registran las señales de aprendizaje (SPEC_LEARNING_SIGNALS). El mecanismo de análisis y actualización automática del perfil es trabajo de Sprint 4 en editorial-profile-dev + writing-dev + evaluation-dev.

## RATIONALE

Un perfil que el editor actualiza manualmente tiende a quedarse desactualizado o a crecer de forma incoherente. El sistema tiene acceso a señales mucho más precisas que cualquier autodescripción del editor: lo que reescribe, lo que conserva, lo que añade que el sistema no propuso. Esas señales, analizadas sistemáticamente, producen un perfil más fiel que uno construido por introspección.

## AFFECTED SUBSYSTEMS

- WRITING: PROMPT_QA_IDEAS captura señales; PROMPT_WRITE_POST registra POST_FINAL_EDITED; SPEC_LEARNING_SIGNALS define el formato
- EDITORIAL: TEMPLATE_EDITOR_PROFILE necesitará campo de historial de actualizaciones — Sprint 4
- EVALUATION: PROMPT_EVALUATE_POST registra señales de aprendizaje en output LEARNING_SIGNALS
