# DECISION LOG ENTRY: DL_20260411_WRITING_016

## DECISION

El POST_SEED es el artefacto canónico que unifica la interfaz de PROMPT_WRITE_POST independientemente del camino de entrada. Combina el contenido procesado (fuentes resumidas, afirmaciones verificadas) con la voz posicionada del editor (output del Q&A, formulaciones literales marcadas). PROMPT_WRITE_POST siempre recibe un POST_SEED, nunca material sin procesar.

## RATIONALE

Sin un artefacto canónico de entrada, PROMPT_WRITE_POST necesitaría gestionar la asimetría entre un input de RAMA POST autónomo y un input procedente de Activation, cada uno con estructura y contenido distintos. El POST_SEED elimina esa asimetría: sea cual sea el camino, el prompt siempre recibe el mismo tipo de artefacto con la misma estructura garantizada.

## AFFECTED SUBSYSTEMS

- WRITING: PROMPT_WRITE_POST recibe siempre POST_SEED como input canónico
- ACTIVATION: El flujo de Activation produce un POST_SEED antes de invocar PROMPT_WRITE_POST
