# DECISION LOG ENTRY: DL_20260411_WRITING_024

## DECISION

PROMPT_WRITE_POST se actualiza de v1.0 (preexistente no documentada formalmente) a v2.0. El input canónico pasa a ser POST_SEED — artefacto estructurado que unifica la interfaz del prompt independientemente del camino de entrada (RAMA POST autónomo o Activation). El modo híbrido se activa solo por declaración explícita del editor. El material citable literal del INVENTARIO_IDEAS se usa sin modificación.

## RATIONALE

Sin POST_SEED como input canónico, PROMPT_WRITE_POST necesitaba gestionar la asimetría entre un input de RAMA POST (material procesado por toda la cadena) y un input de Activation (POST_PLAN con estructura distinta). El POST_SEED elimina esa asimetría: sea cual sea el origen, el prompt siempre recibe el mismo tipo de artefacto con la misma estructura garantizada. El modo híbrido por declaración explícita evita que el sistema modifique texto del editor que no estaba destinado a ser completado.

## AFFECTED SUBSYSTEMS

- ACTIVATION: el flujo de Activation debe producir un POST_SEED antes de invocar PROMPT_WRITE_POST. PROMPT_WRITE_POST ya no acepta un POST_PLAN directo como input.
