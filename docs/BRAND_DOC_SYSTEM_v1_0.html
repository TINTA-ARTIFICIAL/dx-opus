01

Propósito de este documento

Este documento define el sistema visual y tipográfico oficial de toda la documentación del sistema D-X-OPUS. Es la fuente única de verdad para colores, tipografías, componentes y estructura de cualquier documento producido bajo la marca Tinta Artificial.

Cualquier documento del paquete de Release 1 —guías de usuario, specs de subsistema, notas de release o roadmap— debe seguir los estándares aquí definidos sin excepción.

La coherencia visual no es estética: es credibilidad editorial. Un sistema que produce libros con rigor debe documentarse con el mismo rigor.

04

Componentes documentales

Callouts

Tres variantes según el tipo de mensaje. El borde izquierdo de 3px determina la variante.

Callout informativo (Cian). Para notas conceptuales, definiciones clave o contexto importante que el lector debe retener.

Callout de precaución (Ámbar). Para advertencias de deuda técnica, comportamientos temporales o elementos pendientes de revisión.

Callout de alerta (Rojo). Para breaking changes, incompatibilidades o errores conocidos que afectan al flujo.

Code blocks

Para referencias de artefactos, comandos, IDs o secuencias de prompts.

WORKFLOW_RESEARCH → PROMPT_CREATE_RESEARCH_PLAN v3.2 → PROMPT_SUMMARIZE_REFERENCES v4.1 → PROMPT_EVALUATE_RESEARCH_REPORT v1.1

Badges de estado y tipo

Se usan en tablas de inventario, cabeceras de artefacto y listas de tareas.

ACTIVE CRITICAL PENDING DRAFT DEPRECATED SCHEMA WORKFLOW PROMPT RESOURCE TOOL

Tablas

Encabezados en JetBrains Mono mayúsculas, separados del cuerpo por una regla Cian de 1.5px. Filas con separador Papel de 0.5px.

06

Reglas de uso y anti-patrones

Sí — Hacer siempre

Usar Bebas Neue para todos los títulos de sección H1 y H2. Aplicar la regla cian de 2px inmediatamente debajo de cada H1. Usar JetBrains Mono para cualquier referencia a un ID de artefacto, versión o campo de metadata. Escribir el cuerpo siempre en Gentium Plus. Incluir metadata YAML completa en todo artefacto del repositorio.

No — Evitar siempre

No usar Arial o fuentes del sistema para títulos visibles al lector final. No aplicar el color Rojo fuera de contextos de alerta o error. No omitir la regla cian en los H1. No mezclar Gentium Plus y Bebas Neue en el cuerpo de texto. No crear variantes de badge con colores ad hoc fuera de los ocho definidos. No usar cursiva para metadatos técnicos (exclusivamente JetBrains Mono).

Los documentos generados por IA deben aplicar esta plantilla antes de su revisión humana. La conformidad visual es parte del proceso de QA documental.
